Amazon S3 is one of the main building blocks of AWS, and many websites utilize an S3 due to its "infinitely scaling" storage capability.

An S3 essentially allows people to store [[Objects|objects]] (files) in "buckets" (directories). Buckets are defined at a region level. It is best practice when creating buckets that you enable [[Versioning|versioning]] on the bucket, and if you'd like you can also enable [[S3 Replication]].

S3 Buckets can have different levels of security attached to them. There is User-Based security that is controlled through [[IAM]] Policies, and Resource-Based Policies that are set on the bucket level. Buckets can also be encrypted in Amazon S3 using encryption keys

For generating policies like the one pictured above (including policies for other services) Amazon has an AWS Policy Generator that you can use. 

Policies **MUST** be applied to an S3 bucket in order for users to access a statically hosted website on the S3. This is because public access is disabled by default, so a policy must be set to allow entry. If public access is blocked a user will see a `403 Forbidden` error.

S3s, like some other services, have a wide variety of [[S3 Storage Classes|storage options]].
